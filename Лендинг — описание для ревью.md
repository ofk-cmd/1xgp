# Лендинг 1xBet — копия страницы Google Play (KG + TJ)

Документ для быстрого ревью. Лендинг имитирует страницу приложения в Google Play
и служит для раздачи APK по прямой ссылке. Кнопка «Установить» ведёт на
трекинговую CRM-ссылку, **собранную динамически из UTM-меток в URL**.

> Почему фейк-Play: gambling-приложения забанены в Google Play, поэтому APK 1xBet
> (`org.xbet.client1`) раздаётся мимо стора. Страница-копия Play — стандартный способ дистрибуции.

## 2. Как работает кнопка «Установить» (ядро)

CRM выдаёт ссылку вида:
```
https://reffpa.com/L?tag=...&pb=...&click_id={clickid}
```
Чтобы кнопка вела на неё, к домену лендинга добавляются те же значения в одноимённые UTM:
```
https://<домен-лендинга>/?domain=reffpa.com/L&tag=...&pb=...&click_id={clickid}
```
JS на лендинге парсит query и **реконструирует исходную CRM-ссылку**:
```
https://reffpa.com/L?tag=...&pb=...&click_id={clickid}
```

**Критично:** `tag` (содержит `[]`), `pb`, `click_id` (содержит макрос `{clickid}`)
передаются **СЫРЫМИ, без url-кодирования** — иначе трекинг сломается (макрос `{clickid}`
подменяется источником трафика на реальный id). Декодируется только `domain` (один раз).
Параметр `geo` (если есть) — служебный, в ссылку не попадает.

**DEFAULT_URL (заглушка):** если домен/тег слетел или вход кривой (нет меток, нет `domain`,
домен с пробелом или без точки, мусор, ошибка парсинга) → кнопка ведёт на `DEFAULT_URL`.
Логика в try/catch + валидация (`new URL`). Текущее значение:
`https://refpa86112.pro/L?tag=s_5729865m_1524c_&site=5729865&ad=1524` (в генераторе → `COMMON["fallback"]`).

Код (в каждом HTML, в конце, без зависимостей):
```js
var DEFAULT_URL = "https://refpa86112.pro/L?tag=s_5729865m_1524c_&site=5729865&ad=1524";
function buildTarget(){
  try{
    var qs = (location.search||"").replace(/^\?/, "");
    if(!qs) return DEFAULT_URL;
    var parts = qs.split("&"), domain = "", rest = [];
    for(var i=0;i<parts.length;i++){
      var p = parts[i]; if(!p) continue;
      var eq = p.indexOf("="), key = eq>=0 ? p.slice(0,eq) : p;
      if(key==="domain"){ domain = decodeURIComponent(p.slice(eq+1)); }
      else if(key==="geo"){ /* пропускаем */ }
      else { rest.push(p); }            // сырой кусок — без перекодирования
    }
    domain = (domain||"").trim();
    if(!domain || /\s/.test(domain) || domain.indexOf(".")===-1) return DEFAULT_URL;
    var url = "https://" + domain + (rest.length ? "?" + rest.join("&") : "");
    new URL(url);                       // финальная проверка
    return url;
  } catch(e){ return DEFAULT_URL; }
}
```
Ссылка проставляется во **все 3** кнопки (`[data-install]`): основная, sticky-бар, поп-ап.

**Favicon** — `favicon_v3.ico` вшит инлайном в `<head>` как data-URI (читается из `favicon.ico`
при сборке; self-contained, отдельный файл деплоить не нужно).

## 3. Поп-ап с оффером

Лёгкий bottom-sheet (чистый CSS/JS, без библиотек). Показывается **через 2 минуты**
на странице, **один раз**. Оффер — по гео; CTA «Установить и забрать» ведёт на ту же
динамическую ссылку. Закрытие — по ✕ или тапу вне карточки.
Задержка меняется в HTML/генераторе: `POPUP_DELAY_MS = 120000`.

рейтинг, тексты, ЧМ-2026) — в `COMMON` и шаблоне.

**ЧМ-2026** добавлен текстом (в «Об этом приложении», «Что нового», чип, поп-ап) —
без логотипов FIFA по регуляторике.

## 5. Ассеты

Сейчас стоят **официальные** ассеты из `mobile assets/`, оптимизированные под веб:
- иконка `Logo 1024x1024.png` → `assets/icon.png` (144px, 14 КБ)
- скриншоты `ru/6_9_screen_1..6.png` → `assets/screen1..6.webp` (640px, WebP q80, ~40–70 КБ каждый)

**Заменить ассеты:** положить новые файлы в `PlayMarket_Landing/assets/` под теми же
именами (`icon.png`, `screen1..6.webp`). Оригиналы (1024px иконка, скрины 3 размеров) —
в этой папке `mobile assets/`. При желании пересжать — есть Pillow-логика в истории сборки
(ресайз до 640px ширины + WebP q80).

Вес страницы: HTML ~25 КБ + иконка 14 КБ + 6 скринов ~318 КБ. Скриншоты, кроме первого,
грузятся лениво (`loading="lazy"`) → быстрый первый экран.

## 6. Сборка и деплой

**Пересборка** (после правок в `build_playmarket.py`):
```
cd Files/PlayMarket_Landing && python3 build_playmarket.py
```

**Деплой:** на KG-домен заливаем `1xbet_kg.html` (как `index.html`) **+ папку `assets/`**;
на TJ-домен — `1xbet_tj.html` + `assets/`. Трекинговую ссылку строим от своего домена (см. §2).
Стоит `<meta robots noindex,nofollow>` — страница не индексируется.

## 8. Ограничения / заметки

- Лендинг — статика. Реальная логика скачивания APK и постбэки — на стороне CRM/трекера,
  не лендинга.
- Скриншоты — официальные маркетинговые креативы 1xBet (RU). Если нужны KG/TJ-специфичные —
  заменить файлы в `assets/`.
- Лиц-плашку (TOJ TURON для TJ) на странице Play обычно не показывают — оставлено чисто;
  при требовании legal добавляется в подвал.
- 1xBet нет в официальном Google Play — клонируется chrome обычного спорт-листинга.
