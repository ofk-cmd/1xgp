# Трекинговая ссылка на кнопке «Установить»

## Как это работает

На лендинге все кнопки **«Установить»** помечены атрибутом `data-install-link`.  
Скрипт `js/tracking.js` читает параметры из **URL самого лендинга** и собирает итоговую ссылку для CRM.

## Что приходит из CRM

Пример ссылки из CRM:

```
https://reffpa.com/L?tag=d_4653081m_97c__[]general[]_d170276_l215579_other&pb=34c5a465282440ebba6927d791abc868&click_id={clickid}
```

Составляющие:

| Элемент | Значение |
|--------|----------|
| Домен + путь | `reffpa.com/L` |
| Тег | `tag=d_4653081m_97c__[]general[]_d170276_l215579_other` |
| Постбек | `pb=34c5a465282440ebba6927d791abc868` |
| Кликайди | `click_id={clickid}` |

## Как собрать URL лендинга

К домену лендинга добавьте те же значения в одноимённые GET-параметры:

```
https://1xbt.kz/?domain=reffpa.com/L&tag=d_4653081m_97c__[]general[]_d170276_l215579_other&pb=34c5a465282440ebba6927d791abc868&click_id={clickid}
```

## Что получит пользователь на кнопке

При клике на **«Установить»** откроется ссылка, эквивалентная CRM:

```
https://reffpa.com/L?tag=d_4653081m_97c__[]general[]_d170276_l215579_other&pb=34c5a465282440ebba6927d791abc868&click_id={clickid}
```

## Обязательные параметры

- `domain` — домен и путь оффера **без** `https://` (например `reffpa.com/L`)
- `tag` — значение тега из CRM
- `pb` — постбек из CRM

## Необязательные параметры

- `click_id` — идентификатор клика (можно передать плейсхолдер `{clickid}` для подстановки трекером)

## Если параметров нет

Кнопка «Установить» будет неактивной (`href="#"`), клик не уведёт с лендинга.

## Где стоят кнопки с трекингом

На странице одна кнопка **«Установить»** в блоке приложения (на десктопе и мобильном).  
Кнопки «Поделиться» и «Добавить в список желаний» — декоративные, трекинг на них не вешается.
